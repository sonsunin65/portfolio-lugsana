# คู่มือการนำโค้ดขึ้น GitHub และเชื่อมต่อ Vercel (สำหรับผู้เริ่มต้น)

คู่มือนี้จะอธิบายขั้นตอนการนำ Source Code ทั้งหมดขึ้นสู่ GitHub เป็นครั้งแรกแบบละเอียด โดยใช้หน้าจอ Command Prompt (cmd) บน Windows

## สิ่งที่ต้องเตรียม
1.  **บัญชี GitHub**: สมัครได้ที่ [github.com](https://github.com/)
2.  **โปรแกรม Git**: ต้องติดตั้ง Git ในเครื่องแล้ว (เช็คโดยพิมพ์ `git --version` ใน cmd)
3.  **Visual Studio Code (VS Code)**: สำหรับเปิดแก้ไขโค้ด

---

## ส่วนที่ 1: การอัปโหลดครั้งแรก (First Upload)

### ขั้นตอนที่ 1: สร้าง Repository ใหม่บน GitHub

1.  เข้าสู่ระบบ GitHub
2.  ไปที่มุมขวาบน กดเครื่องหมาย **+** แล้วเลือก **New repository**
3.  **Repository name**: ตั้งชื่อโปรเจกต์ภาษาอังกฤษ (เช่น `teacher-portfolio`)
4.  **Description**: ใส่รายละเอียด (ไม่บังคับ)
5.  **Public/Private**: เลือก Public (สาธารณะ) หรือ Private (ส่วนตัว)
6.  **สำคัญ**: ในส่วน **Initialize this repository with:** **ไม่ต้องติ๊กเลือกอะไรเลย** (ปล่อยว่างไว้ เพื่อให้เป็น Repo เปล่าๆ)
7.  กดปุ่ม **Create repository** สีเขียว
8.  เมื่อสร้างเสร็จ คุณจะเห็นหน้าที่มีลิงก์ Repo (เช่น `https://github.com/ชื่อคุณ/teacher-portfolio.git`) **ให้ก๊อปลิงก์นี้เก็บไว้**

### ขั้นตอนที่ 2: เตรียมไฟล์ในเครื่อง (ผ่าน Command Prompt)

1.  กดปุ่ม **Windows** บนคีย์บอร์ด พิมพ์คำว่า `cmd` แล้วกด Enter เพื่อเปิดหน้าจอสีดำ
2.  **สำคัญ**: พิมพ์คำสั่งเพื่อเข้าไปยังโฟลเดอร์ที่เก็บไฟล์งานของคุณ
    *   **วิธีที่ง่ายที่สุด**: พิมพ์คำว่า `cd` (เคาะวรรค 1 ที) จากนั้นกลับไปหน้าโฟลเดอร์งานของคุณ **ลากโฟลเดอร์** มาวางใส่ในหน้าจอดำ แล้วกด Enter
    *   หรือพิมพ์คำสั่งระบุตําแหน่งเอง (ตัวอย่าง):
        ```cmd
        cd c:\Users\Krisada\Documents\cute-teacher-portfolios-main
        ```
    *(หมายเหตุ: ต้องแน่ใจว่า path ใน command line ชี้ไปที่โฟลเดอร์งานของคุณถูกต้องแล้ว ก่อนทำข้อต่อไป)*

**พิมพ์คำสั่งทีละบรรทัดตามลำดับ:**

1.  เริ่มต้นระบบ Git (ทำครั้งแรกครั้งเดียว):
    ```cmd
    git init
    ```

2.  เปลี่ยนชื่อ Branch (ทำครั้งแรกครั้งเดียว):
    ```cmd
    git branch -M main
    ```

3.  เลือกทุกไฟล์เตรียมอัปโหลด:
    ```cmd
    git add .
    ```

4.  บันทึกและใส่ข้อความกำกับ:
    ```cmd
    git commit -m "First upload"
    ```

5.  เชื่อมต่อกับ GitHub (ใส่ลิงก์ของคุณที่ก๊อปมา):
    ```cmd
    git remote add origin https://github.com/ชื่อคุณ/ชื่อโปรเจกต์.git
    ```

6.  ส่งไฟล์ขึ้น GitHub:
    ```cmd
    git push -u origin main
    ```

## ส่วนที่ 2: การอัปเดตเว็บไซต์ภายหลัง (Update Website)
เมื่อคุณมีการแก้ไขโค้ด ปรับแก้คำผิด หรือเปลี่ยนรูปภาพ และต้องการอัปเดตหน้าเว็บ ให้ทำตามขั้นตอนดังนี้ (ใช้คำสั่งเดิมได้เลยครับ):

1.  เปิด **cmd** และใช้คำสั่ง `cd` เข้าไปที่โฟลเดอร์งานเหมือนเดิม
2.  พิมพ์คำสั่ง 3 บรรทัดนี้:

    **บรรทัดที่ 1 (เลือกไฟล์ที่แก้ไขทั้งหมดมาเตรียมไว้):**
    ```cmd
    git add .
    ```
    *(การใช้ `git add .` ปลอดภัยครับ มันจะฉลาดพอที่จะรู้ว่าไฟล์ไหนเปลี่ยน ไฟล์ไหนใหม่ หรือไฟล์ไหนถูกลบ มันจะจัดการให้อัตโนมัติ)*

    **บรรทัดที่ 2 (บันทึกพร้อมบอกว่าแก้อะไรไป):**
    ```cmd
    git commit -m "Update footer text"
    ```
    *(เปลี่ยนข้อความในฟันหนู เป็นอะไรก็ได้ที่สื่อว่าเราแก้อะไรไป เพื่อกันลืม)*

    **บรรทัดที่ 3 (ส่งขึ้นฟ้า):**
    ```cmd
    git push
    ```

รอสักครู่ Vercel จะรู้ตัวอัตโนมัติและทำการอัปเดตหน้าเว็บให้เองครับ

## ส่วนที่ 3: เชื่อมต่อ Vercel (ทำครั้งแรกครั้งเดียว)
**(ถ้าทำไปแล้วไม่ต้องทำซ้ำ)**

1.  ไปที่ [vercel.com](https://vercel.com/)
2.  กด **Add New ...** > **Project**
3.  เลือก Repository ที่เพิ่งอัปโหลดไป กด **Import**
4.  ไปที่ **Environment Variables** ใส่ค่า `VITE_SUPABASE_URL` และ `VITE_SUPABASE_PUBLISHABLE_KEY`
5.  กด **Deploy**
